项目介绍

智慧学堂

我们这个项目是给一个在线教育公司做的，相当于一个平台, 允许全国各地的老师入驻到我们这个平台,上传他们的课程，同样学生也可以来我们的平台进行购买课程。

主要分为那三个客户端？

这个项目主要分为三个用户终端，一个是平台的后台管理，还有老师的后台管理系统和学生端。

都采用了那些技术和中间件？

这个项目采用的是SpringCloudAlibaba做的微服务架构。

使用nacos做为注册中心。

微服务项目之间通信的时候用的是那个openFeign进行通信的。

服务的熔断我们采用的是Sentinel（三特 No）做的熔断。

我们项目中的持久层用的是MyBatisPlus。

为了解决高并发的问题，项目中还用了挺多的互联网项目中经常用到的解决高并发的中间件，比如说redis，ElasticSearch，RabbitMQ，多线程线程池这些技术。

主要有那几个模块？

我们这个项目里主要包含了课程管理，交易查询，优惠券管理,评价管理,课程的搜索功能，课程的详情展示功能，还学生的课程收藏清单，支付，老师结算，大概就这些个模块。

我负责了什么模块？

我负责的模块是：课程管理模块，搜索模块，订单模块，您看你想了解哪一块，我给说一下。

1.【课程管理模块】

​ 首先先说一下课程管理模块，这个模块是整个平台的核心之一，平台这一块可以管理所有老师的课程，可以对老师提交上来的课程进行审核，也可以删除；老师可以上传课程，也可以对自己上传的的课程进行管理,课程审核通过以后就不能删除和编辑了。

我们这个功能模块里的表差不多有个三四张表，课程分类表，课程标题,课程详情这些个表，这个功能里面用到的技术,管理图片的时候，我们用的是FastDFS文件服务器，视频流文件用的是阿里云,大概这些个技术。

我的任务就是设计和实现课程的管理流程，包括课程信息的录入、编辑、状态管理等。我需要确保课程的发布和撤销是稳定和可靠的。主要是通过 RabbitMQ发送⼀条消息，搜索服务⾥有⼀个监听可以监听到发送的信息，然后同步到我们的索引库⾥⾯。

2.【搜索模块】

​ 搜索框的搜索功能允许用户 根据输入的检索条件 查找相关商品，一般来说大多数用户购物的体验流程，就是：搜索想要买的商品 — 选择商品 — 进入商品详情 – 加入购物车 – 下单 – 支付。所以搜索的需求 一定是访问比较高频的 ，按照我们优化的思想 这样的需求 就不能直接查询数据库了 否则数据库的压力过大 会导致宕机。为了高效处理大量用户在搜索框搜索的请求，这个项目选择了 Elasticsearch（伊拉斯怼可 sir器）作为搜索引擎，它采用了倒排索引技术，能够快速找到所需内容。倒排索引的搜索流程是先分词，再查找包含词条的文档ID，最后通过ID在正向索引中查找具体文档。

​ 具体的实现就是使用 Elasticsearch 的 Java 客户端创建一个索引，用于存储商品数据。指定每个字段的数据类型和属性，如课程名称、分类等。创建一个 类，用于表示用户的搜索参数，例如课程关键字、分类 ID等。这个类中的属性需要对应用户搜索时提交的参数。使用 ES 的 API ，根据用户传递的参数构造查询 DSL。DSL 中包括匹配课程名称的查询、按分类 ID 进行过滤的查询等。将构建好的查询 DSL 发送给 ES的Java 客户端执行查询。ES 将返回搜索结果，其中包含符合搜索条件的商品数据。最后把这些数据处理一下，返回给前端。

3.【订单模块】

​ 然后是我负责的这个订单模块，我的任务就是设计并实现一个稳定的订单管理系统，保证学生可以顺利选课并支付，同时也能够为老师提供准确的结算信息。我首先定义了订单的数据结构，然后使用 MybatisPlus 进行数据库操作。我开发了选课的流程，确保学生可以将课程添加到购物车、选择支付方式，结算并生成订单。结算的时候要验证价格，确保学生在支付时是最新的价格，这方面是通过调用商品服务的接口来实现的，在订单服务中，定义了一个方法，用于验证订单中每个商品的价格是否与当前系统中的价格一致。该方法接收课程ID和订单中填写的价格作为参数，返回一个布尔值，表示价格是否一致。如果两个价格相等，则进行支付，如果价格不一致，则订单服务会返回失败信息，告知用户该商品的价格有变动，无法下单，请刷新一下再试。最后确保订单不重复提交是通过生成并验证结算流水号来实现的。

港富银行

我们这个项目是给地方银行做的，是一个综合管理平台，是为了解决数据实时更新耗费大量人力、物力和时间的问题。

我们这个项目分为web管理端和移动端两个端口，WEB管理端主要模块包含：系统管理、商户管理、设备管理、工单任务管理、数据统计分析。移动端主要模块包含：用户管理、工单任务分发和审核、工单任务的回执。大概就是这些个模块了。

我们采⽤ SpringBoot、SpringMVC、Mybatis做的技术栈、项⽬中为了解决⾼并发还⽤到了很多的中间件，⽐如说 redis，ElasticSearch、RabbitMQ，多线程线程池这些技术。

对于管理系统而言，大量的图片是必不可少的，使⽤ FastDFS 来管理图⽚，通过使⽤ FastDFS 搭建图⽚存储服务器，减少服务器压力。

这个项目主要有：用户管理模块、信用卡模块、支付与转账的模块、报表管理模块、存款与贷款模块。

我主要负责的模块有：支付与转账的模块、报表管理模块、存款与贷款模块这三个模块。您看你想了解哪一块，我给您细说一下。

【支付与转账的模块】

  1. 第三方支

集成支付宝和微信支付等第三方支付系统，使用户可以选择这些支付方式进行转账和汇款。

  1. 转账

       首先我们要确保付款方账户有足够的资金来支付转账金额。后端接收到用户在界面上输入付款金额、付款方账户、收款方账户等信息，验证后，根据付款方账户，查询数据库以获取其当前余额。检查付款方余额是否大于等于转账金额。如果余额不足，终止转账流程并通知用户。如果余额足够，计算并更新付款方账户余额。将扣除后的余额更新到数据库中。确保收款方账户正确地接收转账金额。根据收款方账户，查询数据库以获取其当前信息。将转账金额添加到收款方账户余额中。更新数据库中的收款方账户信息。为了确保整个转账过程的数据一致性，应该使用事务管理。在执行付款方扣款和收款方入账操作时开启数据库事务。这确保了在一系列操作中，保证两个操作都成功才提交事务，如果任何一个操作失败，回滚事务，恢复到操作前的状态，通知用户转账失败的原因。在转账过程中，可能会出现各种异常情况，需要进行适当的处理。一般都是 数据库异常或网络异常。根据异常类型，执行适当的回滚或处理策略。在转账操作后，立即向用户展示操作结果，成功或失败。

2.【报表管理模块】

报表管理模块这个模块用到了四张表，分别是新增客户报表、新增商户报表、新增终端报表。因为银行对于数据的存储是十分重视的，所以我们在报表这块我们用到了cogons（扣苟）来提取银行系统的数据，然后分析并汇总得出报告。

jasperreport（加死班 锐泡尔特）报表引擎，使报表的布局变的灵活，从而从多方面来提供数据，进行文件转换。数据仓库来存储大量的数据，同时从事务系统获取数据，并做汇总、加工，主要为了给决策者提供决策的依据。因为事务系统是实时的，这就要求时效性，如果客户存一笔钱需要几十秒速度就太慢了，这就要求数据库只能存储很短一段时间的数据。而分析系统是事后的，它要提供关注时间段内所有的有效数据。这些数据量比较大，汇总计算起来也要慢一些，但是，只要能够提供有效的分析数据就达到目的了。etl工具来进行数据的同步，大数据量的整合，为数据仓库工程提供了很大的便利。报表导出大量的excel（依克搜）文件我用的是poi框架，图表这块，我们是在js层用的highchart（嗨 char 特）组件，大概就是用了这些个技术。

报表中的具体字段：

新增客户报表：

​ 客户名称、客户号、入驻时间、联系人、联系电话

新增商户报表：

​ 商户中文简称、商户号、入驻时间、联系人、联系电话、商户地址

新增终端报表：

​ 商户中文简称、商户号、终端号、终端入驻时间、终端类型

3.【存款与贷款模块】

当负责存款与贷款模块时，你需要实现存款产品管理、贷款产品管理以及贷款申请与审批。以下是详细的逻辑和思路：

  1. 存款产品管理

存款产品是银行为客户提供的一种理财服务，用户可以将资金存入账户并获得利息。首先，我们提供了一些不同种类的存款产品，例如活期存款、定期存款、储蓄存款等。每种存款产品可能具有不同的特点，如存款期限、利率、提前支取条件等。我们还设计一个数据库表，用于存储存款产品的详细信息。表中的字段可以包括存款产品名称、利率、期限、最小存款金额、提前支取利率等信息。

具体的流程是：用户在登录到银行系统后，可以浏览不同种类的存款产品并选择感兴趣的产品类型和存款金额。一旦用户选择了存款产品，系统可以根据该产品的利率和期限计算出存款期限内的预期利息。这可以通过利息计算公式来完成，同时也考虑到复利计算。在存款操作完成后，系统将用户选择的存款金额加上计算得出的利息总额存入用户的账户。同时，系统还需要记录存款的详细信息，如存款时间、产品类型等。

  1. 贷款产品管理

贷款产品管理是银行和金融机构中重要的服务之一，涉及到贷款产品的定义、创建、申请以及还款计划的计算。

首先，我们提供了一些不同种类的贷款产品，如个人消费贷款、房屋贷款、教育贷款等。每种贷款产品可能具有不同的特点，如贷款利率、贷款额度、还款期限等。然后是设计一个数据库表，用于存储贷款产品的详细信息。表中的字段可以包括贷款产品名称、利率、最大贷款额度、最长还款期限、提前还款政策等信息。

具体的流程是：用户登录到银行系统后，可以查看不同种类的贷款产品，选择适合自己需求的贷款类型和贷款金额。用户还需要填写贷款申请所需的个人信息，如收入、职业等。一旦用户提交贷款申请，系统可以根据用户提供的信息和所选贷款产品的特点，计算出合适的还款计划。还款计划可能包括每期还款金额、还款日期、还款利息等详细信息。

贷款产品管理系统还可以包括以下功能：

  • 贷款申请审批： 银行工作人员可以登录后台管理界面，审查用户的贷款申请信息，并根据 一定的审批流程决定是否批准贷款。

  • 贷款合同生成： 一旦贷款申请获得批准，系统可以生成贷款合同，其中包括贷款金额、利 率、还款期限等详细条款。

  • 还款管理： 系统可以跟踪和管理用户的还款情况，包括记录每次还款的金额和日期，并计 算剩余未还本金和利息。

  • 提前还款处理： 用户有可能选择提前还款，系统可以根据提前还款政策计算提前还款金额， 并更新还款计划。

  • 通知和提醒： 系统可以发送通知和提醒，提醒用户还款日期、还款金额等重要信息。

  1. 贷款申请与审批

贷款申请与审批是贷款服务过程中的核心环节，涉及用户提交贷款申请、银行进行审批，以及最终贷款发放和还款的过程。

用户登录到银行系统，选择所需的贷款产品，并提供必要的个人信息、财务信息等。这些信息可能包括个人身份、收入、职业等。银行系统会对用户提交的信息进行验证和处理，确保其完整性和准确性。一旦信息验证通过，系统将生成贷款申请记录，开始后续的审批流程。

银行工作人员可以使用后台管理界面查看待审批的贷款申请列表，了解申请的情况。然后进行审核，包括对用户的信用评估、还款能力、财务状况等进行评估。这可能涉及内部数据查询、信用报告的获取等。一旦审批完成，银行会向用户通知审批结果。如果贷款申请被批准，银行会生成贷款合同，其中包括贷款金额、利率、还款期限、还款计划等详细条款。

一旦用户同意并签署贷款合同，合同即生效。银行会将贷款金额划入用户指定的账户，用户可以使用这笔资金。根据贷款合同中规定的还款计划，用户需要按时进行还款。银行会提供相应的还款渠道和方式，用户可以选择合适的方式进行还款。

4.【参与后台管理模块】

  • 设计管理员界面，允许管理员进行不同的操作，如设置配置、管理用户等。

  • 实现界面与后端逻辑的交互，确保管理员操作能够正确地更新数据库。

  • 设计数据库表，存储系统设置、用户信息、账户信息等。

  • 在系统中引入权限系统，确保只有管理员可以访问后台管理功能。

  • 实现审计日志记录功能，将关键操作记录到数据库，以便追踪和分析。
